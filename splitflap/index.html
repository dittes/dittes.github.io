How to pair
1

Open both pages

Open Display on your TV/monitor and Controller on your phone. They can be on different devices.

2

Generate invite code on Display

Click "Pair" in the header → "Generate Invite Code." Wait for the code, then copy it.

3

Paste into Controller

Paste the invite code into the controller and click "Generate Response." Copy the response code.

4

Paste response into Display

Paste the response code into the display's step 2 field and click "Connect." You're live.

Good to know

Do I need a server?

No. Host these three files anywhere — GitHub Pages, Netlify, Cloudflare Pages, or even open them from your hard drive.

How does it work without a server?

WebRTC peer-to-peer connection. The invite/response codes replace the signaling server with a manual copy-paste step.

Will it work across networks?

Uses Google's public STUN servers for NAT traversal. Works on most networks. Very restrictive firewalls may block it — same network is most reliable.

Best TV setup?

Open display.html in Chrome, click the fullscreen button ⛶ in the header. The display attempts to keep the screen awake.

Can I reload?

Both pages save state to localStorage. After reload the display shows what was last on the board. You'll need to re-pair though.
