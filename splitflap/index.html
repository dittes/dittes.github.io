How to pair
1

Open both pages

Open the Display on your TV/monitor and the Controller on your phone or laptop. They can be different devices on the same network — or even different networks.

2

Generate invite code on Display

Click "Pair" in the Display header, then "Generate Invite Code." Copy the code that appears.

3

Paste into Controller

On the Controller page, paste the invite code and click "Generate Response." A response code will appear.

4

Paste response back into Display

Copy the response code from the Controller and paste it into the Display's "response code" field. Click "Connect." That's it — you're live.

Good to know

Do I need a server?

No. These are three static HTML files. Host them anywhere — a simple file server, GitHub Pages, Netlify, Cloudflare Pages, or even open them directly from disk (file://).

How does it work without a server?

The Display and Controller connect directly via WebRTC — the same technology behind video calls. The "invite code" and "response code" replace the signaling server with a manual copy-paste step. Once connected, all data flows peer-to-peer.

Will it work across different networks?

It uses Google's public STUN servers to handle NAT traversal, which works for most home/office networks. Very restrictive corporate firewalls or symmetric NATs may block it. For best results, use devices on the same Wi-Fi network.

Best setup for a TV?

Open display.html in Chrome on a device connected to the TV (Chromecast, HDMI laptop, smart TV browser). Click the fullscreen button ⛶ in the header. The display will attempt to keep the screen awake automatically.

Can I bookmark and reuse?

Yes. Both pages save their last state to localStorage. After a reload, the display shows whatever was last on the board. You only need to re-pair if you reload.
