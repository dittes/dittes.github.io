How to pair
1

Open both pages

Display on your TV/monitor, Controller on your phone or laptop.

2

Display: Generate invite code

Click "Pair" → "Generate Invite Code." Wait a few seconds for network candidates, then click "Copy Code."

3

Controller: Paste & generate response

Paste the invite code, click "Generate Response." Wait for candidates, then copy the response code.

4

Display: Paste response & connect

Paste the response code into step 2 on the display, click "Connect." Done.

Troubleshooting

Self-test first

Open the display page, click "Pair", then click "Self-Test". This creates a loopback WebRTC connection on the same page to verify WebRTC works at all in your browser. If the self-test passes, the issue is network-related. If it fails, WebRTC is blocked.

ICE goes to "checking" then "disconnected"

This means signaling worked but the two browsers can't reach each other at the network level. Check the debug panel for candidate types. If you only see "host" candidates, STUN failed — check internet connectivity. If you see "srflx" but still fail, the NAT doesn't support hairpinning (try from different devices/networks).

Answer has zero candidates

If the response code contains zero candidates, the controller's WebRTC may be blocked. Try a different browser (Chrome is most reliable) or check that you're on HTTPS.

Best way to test

For quickest testing, open both pages on the same machine in two Chrome tabs. Host candidates (local network) should connect instantly. For cross-device, both need internet access for STUN servers.

Serve over HTTPS

WebRTC requires a secure context. Files hosted on HTTPS work best. For local testing: python3 -m http.server 8080 and open http://localhost:8080. Localhost is treated as secure.
