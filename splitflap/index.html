How to pair
1

Open both pages

Open Display on your TV/monitor and Controller on your phone or laptop.

2

Generate invite code on Display

Click "Pair" → "Generate Invite Code." Wait for the code (a few seconds while network candidates are gathered), then copy it.

3

Paste into Controller & generate response

Paste the invite code into the controller and click "Generate Response." Wait for the response code, then copy it.

4

Paste response into Display & connect

Paste the response code into the display's step 2 field and click "Connect." The connection establishes within seconds.

Troubleshooting

Connection goes to "disconnected" immediately

This usually means the two browsers can't reach each other at the network level. The most reliable fix: serve the files over HTTPS (or localhost). Opening from file:// disables WebRTC in most browsers. Use python3 -m http.server for a quick local server, or deploy to any static host (GitHub Pages, Netlify, Cloudflare Pages).

Works on same machine but not across devices

Both devices need internet access for the STUN/TURN relay servers to work. If behind a corporate firewall or VPN, try using your phone's mobile data instead of Wi-Fi. The system includes TURN relay servers as fallback for restrictive networks.

I see "ICE candidate: host" but no "srflx" or "relay"

This means STUN/TURN servers couldn't be reached. Check internet connectivity. Host-only candidates work when both devices are on the same local network.

Best setup for a TV?

Open display.html in Chrome on a device connected to the TV (HDMI, Chromecast, smart TV browser). Click ⛶ for fullscreen. The display will try to keep the screen awake via the Wake Lock API.

Do I need a server?

Not a backend server — but the files should be served over HTTP/HTTPS for WebRTC to work reliably. Any static file host works: python3 -m http.server, GitHub Pages, Netlify, or Cloudflare Pages.

Can I reload without losing the board?

Yes — both pages save state to localStorage. The display restores its last content on reload. You will need to re-pair though, since WebRTC connections don't persist across page loads.
